# plants-site

Сайт предприятия «Красная гвоздика» с локальным каталогом и админ-панелью на одном хосте.

## Структура

- `index.html` — главная страница
- `catalog.html` — каталог растений
- `about.html` — страница о предприятии
- `info.html` — контакты и справочная информация
- `style.css` — общие стили сайта
- `js/app.js` — точка входа клиентской логики
- `js/catalog-data.js` — загрузка и подготовка данных каталога
- `js/catalog-page.js` — логика страницы каталога
- `js/home-page.js` — логика главной страницы
- `js/site-ui.js` — общие элементы интерфейса
- `js/utils.js` — вспомогательные функции
- `admin/` — каркас админ-зоны
- `api/catalog.php` — локальный JSON API каталога
- `backend/` — серверная PHP-логика
- `data/catalog.json` — локальное хранилище каталога
- `images/catalog/` — каталог изображений, загружаемых через админку
- `docs/architecture.md` — проектная схема перехода с Google Sheets
- `images/` — изображения растений и контента

## Откуда берутся данные каталога

Публичный каталог читает локальный файл `data/catalog.json`.

Логика загрузки находится в `js/catalog-data.js`, а изменение данных выполняется через `admin/` и PHP-код в `backend/`.

## Что уже можно тестировать

1. Публичную часть сайта через `data/catalog.json`.
2. Админку по адресам `/admin/login.php` и `/admin/index.php`, если хост поддерживает PHP.
3. JSON API по адресу `/api/catalog.php`.

Данные админки пишутся в `data/catalog.json`.

Текущий логин в каркасе:

- `admin`
- `admin111`

Перед реальным размещением это обязательно надо заменить.

## Локальный запуск

Фронтенд лучше открывать через простой HTTP-сервер, а PHP-каркас тестировать через локальный PHP server.

Пример:

```bash
python3 -m http.server 8000
```

После этого открыть `http://localhost:8000`.

Для PHP-каркаса:

```bash
php -S localhost:8080
```

После этого можно открыть:

- `http://localhost:8080/admin/login.php`
- `http://localhost:8080/api/catalog.php`
